 1.  DebateState.get() re-introduces the very bug it was meant to fix
================================================================================
FILE     : src/core/models.py
SEVERITY : CRITICAL

PROBLEM:
  The current implementation is:
      def get(self, key, default=None):
          val = getattr(self, key, _MISSING)
          return default if val is _MISSING or val is None else val

  The clause "or val is None" means that any field that is EXPLICITLY set to
  None (e.g. verdict=None before the debate runs, confidence=None at start)
  will return `default` instead of the actual None value.  This corrupts the
  verdict display logic: state.get("verdict","UNKNOWN") returns "UNKNOWN" even
  when verdict has been deliberately set to None.

  The rule should be: only substitute default when the attribute does not exist
  at all (i.e., getattr raised AttributeError, caught by the sentinel).  If
  the attribute exists and its value happens to be None, that is a legitimate
  value and must be returned.

BUGGY CODE:
    def get(self, key, default=None):
        val = getattr(self, key, _MISSING)
        return default if val is _MISSING or val is None else val

FIXED CODE:
    def get(self, key, default=None):
        val = getattr(self, key, _MISSING)
        return default if val is _MISSING else val

NOTE: The same bug exists in AgentResponse.get() which was never touched.
  Fix that one too (same replacement — swap the whole method body).

 4.  call_structured appends rate-limit timestamp inside the except block —
     JSON parse failures inflate the rate counter
================================================================================
FILE     : src/llm/client.py  (call_structured, both Groq and Gemini blocks)
SEVERITY : CRITICAL

PROBLEM:
  Current code structure (shown for Groq; Gemini is identical):

      try:
          with self._counter_lock:
              if not self._check_rate_limit(...): raise ...   # ← check
          response = self._call_groq(...)                     # ← API call
          result = output_schema.parse_raw(response)          # ← parse
          with self._counter_lock:
              self.groq_last_call_times.append(time.time())   # ← on success only
          return result
      except Exception as e:
          with self._counter_lock:
              self.groq_last_call_times.append(time.time())   # ← on ALL failures

  Problem A: The timestamp is appended TWICE on success (once in try, once
  would fire if try continued past return — actually not, because return exits,
  so on success it is appended once in try, which is fine).

  Problem B: On ANY exception — including output_schema.parse_raw() raising a
  Pydantic ValidationError when the model returns malformed JSON — the except
  block appends a timestamp.  If the model consistently returns invalid JSON
  (common with smaller models), every attempt increments the rate counter even
  though no API slot was actually consumed.  After enough JSON failures the
  limiter starts blocking legitimate calls.

  Problem C: There is a window between the rate-check lock release and the
  actual API call where another thread can pass the same check.

  The correct pattern: append the timestamp unconditionally INSIDE the same
  lock block as _check_rate_limit, BEFORE the API call.  This is atomic and
  counts the API attempt — not the parse result.

BUGGY CODE (Groq block — fix Gemini block identically):
      try:
          with self._counter_lock:
              if not self._check_rate_limit("groq", self.groq_last_call_times):
                  raise RuntimeError("Groq rate limit exceeded")

          response = self._call_groq(
              json_prompt, temperature, max_tokens, timeout=30
          )
          result = output_schema.parse_raw(response)

          with self._counter_lock:
              self.groq_last_call_times.append(time.time())
          return result

      except Exception as e:
          with self._counter_lock:
              self.groq_last_call_times.append(time.time())
          last_error = e
          ...

FIXED CODE (Groq block):
      try:
          with self._counter_lock:
              if not self._check_rate_limit("groq", self.groq_last_call_times):
                  raise RuntimeError("Groq rate limit exceeded")
              # Append timestamp immediately after the check — counts the attempt,
              # not the parse result.  Still inside the lock so it is atomic.
              self.groq_last_call_times.append(time.time())

          if self.groq_client is None:
              raise RuntimeError("Groq client not initialized")

          response = self._call_groq(
              json_prompt, temperature, max_tokens, timeout=30
          )
          result = output_schema.parse_raw(response)
          return result

      except Exception as e:
          # Do NOT append here — the timestamp was already appended above.
          last_error = e
          err_str = str(e).lower()
          if any(kw in err_str for kw in ["429", "rate limit", "quota", "exceeded",
                                           "limit reached", "model_decommissioned"]):
              logger.warning(f"Groq quota/limit error: {e}")
              break
          logger.warning(f"Groq attempt {attempt+1} failed: {e}")
          if attempt < max_retries:
              time.sleep(1)

  Apply the exact same change to the Gemini block inside call_structured.

  Also add the same "append before call" pattern to the call() method where
  Groq and Gemini timestamps are currently appended in the except block:

  In call(), Groq block — change:
      except Exception as e:
          with self._counter_lock:
              self.groq_last_call_times.append(time.time())
          ...

  To: append after the rate check passes and before _call_groq() is invoked
  (same pattern as call_structured fix above).  Apply identically to Gemini.

 5.  _retry_revision_node generates with state.round=4 so agents use the
     wrong prompt branch (rebuttal instead of first-round)
================================================================================
FILE     : src/orchestration/debate.py  (_retry_revision_node)
SEVERITY : CRITICAL

PROBLEM:
  After three debate rounds, state.round == 4 (ConAgent increments it).
  The revision node is triggered after fact_checker_node, which runs at this
  point.  Both ProAgent._build_prompt and ConAgent._build_prompt check:

      if round_num == 1:
          ... build fresh argument ...
      else:
          ... build rebuttal against opponent's latest argument ...

  When revision runs with state.round=4, both agents take the else-branch and
  generate rebuttals to each other's Round 3 arguments.  What we actually want
  is a fresh reformulation of Round 3 with the verification feedback injected.
  The round value should be temporarily set to num_rounds (3) so agents
  generate a rebuttal-style response for the correct round context, then
  restored to 4 after.

BUGGY CODE:
    def _retry_revision_node(self, state: DebateState) -> DebateState:
        logger.info("Revision loop triggered")
        state.retry_count += 1
        # Replace the last entries in-place rather than appending to prevent round bloat
        pro_resp = self.pro_agent.generate(state)
        if state.pro_arguments:
            state.pro_arguments[-1] = pro_resp.argument
            state.pro_sources[-1] = pro_resp.sources

        con_resp = self.con_agent.generate(state)
        if state.con_arguments:
            state.con_arguments[-1] = con_resp.argument
            state.con_sources[-1] = con_resp.sources
        return state

FIXED CODE:
    def _retry_revision_node(self, state: DebateState) -> DebateState:
        logger.info("Revision loop triggered")
        state.retry_count += 1

        # Temporarily set round to the last real debate round so agents build
        # prompts for the correct context (not round 4 which does not exist).
        saved_round = state.round
        state.round = state.num_rounds  # = 3

        pro_resp = self.pro_agent.generate(state)
        if state.pro_arguments:
            state.pro_arguments[-1] = pro_resp.argument
        if state.pro_sources:
            state.pro_sources[-1] = pro_resp.sources

        con_resp = self.con_agent.generate(state)
        if state.con_arguments:
            state.con_arguments[-1] = con_resp.argument
        if state.con_sources:
            state.con_sources[-1] = con_resp.sources

        # Restore the post-increment round value so the rest of the graph
        # sees the correct state.
        state.round = saved_round
        return state

 6.  _verification_gate_node runs on the final round — wasteful HTTP calls
     whose feedback is never read
================================================================================
FILE     : src/orchestration/debate.py  (_verification_gate_node)
SEVERITY : HIGH

PROBLEM:
  The graph is:  con_agent → verification_gate → _should_continue
  After Round 3 con_agent runs, state.round becomes 4.  The gate guard is:
      if not state.round > 1: return state
  4 > 1 is True, so the gate runs, verifies all Round 3 URLs over HTTP (5–15
  seconds), and sets verification_feedback.  _should_continue then returns
  "end", so the feedback is never read by any agent.  Pure wasted latency.

  The gate should only run when there is a subsequent round to inject feedback
  into — i.e., when round_idx < num_rounds - 1 (0-indexed: rounds 0 and 1 of
  a 3-round debate, skip round 2 which is the last).

BUGGY CODE:
    def _verification_gate_node(self, state: DebateState) -> DebateState:
        if not state.round > 1:
            return state
        ...
        round_idx = len(state.pro_sources) - 1
        ...

FIXED CODE:
    def _verification_gate_node(self, state: DebateState) -> DebateState:
        # round_idx is 0-based index of the round just completed.
        # len(state.pro_sources) equals the number of completed rounds.
        round_idx = len(state.pro_sources) - 1

        # Do not run on the very first entry (no sources yet) or on the
        # final round (verification feedback would never be used by any agent).
        if round_idx < 0 or round_idx >= state.num_rounds - 1:
            return state

        logger.info("Running mid-debate verification gate for round %d...", round_idx + 1)

        sources_to_check = []
        if len(state.pro_sources) > round_idx:
            argument = state.pro_arguments[round_idx] if len(state.pro_arguments) > round_idx else ""
            for url in state.pro_sources[round_idx]:
                sources_to_check.append((url, "PRO", argument))
        if len(state.con_sources) > round_idx:
            argument = state.con_arguments[round_idx] if len(state.con_arguments) > round_idx else ""
            for url in state.con_sources[round_idx]:
                sources_to_check.append((url, "CON", argument))

        failed_sources = []
        for url, agent, argument in sources_to_check:
            try:
                verification = self.fact_checker._verify_url(url, agent, argument)
                if verification.status != "VERIFIED":
                    failed_sources.append(url)
            except Exception as e:
                logger.warning(f"Verification gate failed for {url}: {e}")
                failed_sources.append(url)

        if failed_sources:
            state.verification_feedback = (
                "WARNING: The following sources failed verification:\n" +
                "\n".join(f"- {url}" for url in failed_sources) +
                "\n\nYou must revise your argument without these sources."
            )
            logger.info(f"Verification gate caught {len(failed_sources)} failed sources.")

        return state

     checkpoint state on second submission
================================================================================
FILE     : app.py  (analyze_claim_with_streaming, inside the function)
SEVERITY : HIGH

PROBLEM:
  Current code:

      st.session_state.thread_id = f"claim_{hashlib.md5(claim_text.encode()).hexdigest()[:12]}"

  LangGraph's MemorySaver stores checkpoints keyed by thread_id.  If the same
  claim is submitted twice (or the same claim is in the semantic cache miss
  path), the second run picks up the checkpoint from the first run — round
  counters, pro_arguments, con_arguments are already populated.  LangGraph
  skips nodes that are already checkpointed, producing empty or wrong results.

  Cache deduplication is the job of the semantic cache, not LangGraph threads.
  Each submission must get a fresh thread_id regardless of the claim text.

BUGGY CODE:
        # Use hash-based thread_id to ensure same claim = same thread state (Persistence #18)
        import hashlib
        st.session_state.thread_id = f"claim_{hashlib.md5(claim_text.encode()).hexdigest()[:12]}"

FIXED CODE:
        # Always use a fresh UUID so each submission gets a clean LangGraph state.
        # Deduplication of identical claims is handled by the semantic cache in
        # debate.run(), not here.
        st.session_state.thread_id = str(uuid.uuid4())

  uuid is already imported at the top of app.py, so no new import is needed.
